<p><strong>NAT Traversal</strong></p>
<p>a technique to traverse Network Address Translation over UDP socket.</p>
<p>the user/s have to run the &quot;listener.py&quot; script first on the machine that is sitting 1 hop away from its public IP address:</p>
<p>192.168.1.100 &lt;-&gt; 192.168.1.1 &lt;-&gt; 1.1.1.1</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;local &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; gateway &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;public</p>
<p><br></p>
<p>the listener script will start mapping remote addresses to local addresses, assume the other side&#39;s public IP is 2.2.2.2,</p>
<p>the listener will open a 1000 sockets and send a packet on each one to the same remote port, in the router mapping table it will look something like this:</p>

<p><br></p>
<p>the it will start sending the router crafted packets with source address of the remote address 1.1.1.1:5000 and destination address of: 2.2.2.2:randomly-chosen-port</p>
<p>since there are a 1000 mapped entries listening to this address, there is 1000/65535 chance to get a packet through, that is 1.5% but when repeated say 100 times(crafted packets) then most likely one of these packet will get through, once a socket received this packet it will save the socket mapping the &nbsp;public_address : <span style='color: rgb(0, 0, 0); font-family: "Times New Roman"; font-size: medium; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: start; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-style: initial; text-decoration-color: initial; display: inline !important; float: none;'>remote_address, and the all other sockets will close</span></p>
<p><span style='color: rgb(0, 0, 0); font-family: "Times New Roman"; font-size: medium; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: start; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-style: initial; text-decoration-color: initial; display: inline !important; float: none;'>this process is repeated a number of times specified by the user, for a different <strong>remote port&nbsp;</strong>afterwards a number of public address with <strong>known ports</strong> will be mapped to known addresses.</span></p>
